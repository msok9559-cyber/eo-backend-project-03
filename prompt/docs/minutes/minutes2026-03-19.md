# 3차 프로젝트 1팀 회의록
- 날짜 2026년 03월 18일
- 팀명 : Prompt
- 참여자 : 정확,김재웅,박민성
## 오전회의 내용
- 자기소개하기
- 팀장 뽑기 ( 정확씨 당선 )
- 팀명 정하기 (닥트리오)
- 프로젝트 명 정하기 (prompt)
- 깃헙 저장소 복제 및 피그마 ERD 초대 수락
- 작업 요구 명세서 작성
## 작업 목표
- 코드 다듬기
- 관리자 페이지 작성
## 작업 회고
- 관리자 정책 수정 기능을 구현했다.
  Authentication 처리와 로그 저장 구조에서 오류가 발생했으며, 제약조건으로 인해 정책 수정 로직이 실패하는 문제를 해결했다.
  기능의 성격에 맞는 엔티티 설계와 인증 정보 활용의 중요성을 느꼈다.
- 시큐리티가 달라져서 data.sql에 넣어둔 테스트유저가 안되서 계속 고쳐보기 위해 찾아 보는중인데 문제가 하나 있는데 @Bean 등록이 충돌이 있는걸 확인함 일반 유저로그인도 어드민 테이블에서 조회되고 있는 문제를 확인함
## 내일목표
- 플랜 엔티티 쪽 관리자하고 연동 작업 잠금시 표현
- 푹시고 다음주 월요일에 만나기
## 팀 규칙
- ERD, 화면설계서, 명세서 는 기능이 변경 및 추가 될때마다 그때그때 바로 추가하고 팀원들 한테 고지할 것
- 다른사람이 작성하는 파일을 건드리거나 변경시 말하고 작업 시작하기 혹시 누가 작업중일 경우 잠깐 스탑 하고 다른일 먼저 처리
- 주석 달기 / 테스트 잘 실행하고 추가하기
- 야근이을 안하기 위해 최대한 노력하고 혹시나 야근이 필요할경우 야근 참석하기
- 싸우지말기
